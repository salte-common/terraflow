# Example Projects

This directory contains complete example projects generated using `terraflow init` for reference.

## Available Examples

- **[AWS + JavaScript](./aws-javascript/)** - Complete AWS project with JavaScript application code
- **[Azure + TypeScript](./azure-typescript/)** - Complete Azure project with TypeScript application code
- **[GCP + Python](./gcp-python/)** - Complete GCP project with Python application code

## Using These Examples

These examples demonstrate the complete project structure generated by `terraflow init`. You can:

1. **Browse the structure** to understand what files are generated
2. **Reference the configuration** to see how `.tfwconfig.yml` is set up for each provider
3. **Copy patterns** from the Terraform files for your own projects
4. **Study the application code** to see language-specific templates

## Generating Your Own

To generate your own project:

```bash
# AWS with JavaScript
terraflow init my-project --provider aws --language javascript

# Azure with TypeScript
terraflow init my-project --provider azure --language typescript

# GCP with Python
terraflow init my-project --provider gcp --language python

# GCP with Go
terraflow init my-project --provider gcp --language go
```

See the [Project Scaffolding Guide](../scaffolding.md) for complete documentation.

## Example Structure

Each example project includes:

```
<project-name>/
├── src/
│   ├── main/          # Application main files
│   └── test/          # Test files
├── terraform/
│   ├── modules/       # Terraform module templates
│   ├── _init.tf       # Provider and backend configuration
│   ├── inputs.tf      # Input variables
│   ├── locals.tf      # Local values (tags, etc.)
│   ├── main.tf        # Main infrastructure resources
│   └── outputs.tf     # Output values
├── .tfwconfig.yml     # Terraflow configuration
├── .env.example       # Environment variables template
├── .gitignore         # Git ignore rules
└── README.md          # Project documentation
```
